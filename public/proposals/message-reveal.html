03 リサーチ：セクション登場演出 8手法

「見せ場が画面に入る瞬間」を演出する手法。派手すぎず“静かに立ち上がる”ものが本命。

① 実装：IO＋CSS（軽量）

Fade / Fade-up

opacity 0→1＋わずかな translateY。既存 .appear と同系。見せ場では尺を長め（800–1100ms）＋深いease-outで“上質化”。

  • 最軽量・全対応・アクセシブル
  • 単体だと“普通のカード出現”に見えがち
難易度：低GPU◎

IO fade-up

② 実装：IO＋CSS filter

Blur-in（ボケ→シャープ）

filter: blur(10px)→0＋fade。レンズがピントを合わせるような“高級・シネマ感”。食＆温かみの写真に好相性。

  • 安価な工数で一気に上質
  • blurは塗り負荷大。半径≤12px・大面積はモバイル注意
難易度：低〜中塗り負荷△

blur() 性能

③ 実装：IO＋CSS clip-path

Clip-path ワイプ

inset(100% 0 0 0)→inset(0) で下から現れる“めくり”。レイアウトシフト無し・GPU加速。

  • 「シーンが現れる」編集的な一手
  • 硬いワイプは“静か”に反する→遅いeaseとfade併用
難易度：中GPU◎

Emil: clip-path

④ 実装：CSS transform

Scale settle / Ken Burns

登場：scale(1.06→1)＋fadeで“収まる”高級ヒーローの定番。Ken Burns：15–25sの緩い寄りで“生きた写真”。

  • transformのみで軽い／温度感UP
  • 寄せ過ぎは写真（のれん/提灯）が切れる。振幅≤8%
難易度：低GPU◎

Ken Burns CSS

⑤ 実装：scroll()／JS

Parallax（写真だけ遅く）

スクロールで写真がテキストより遅く動き“奥行き”。transform: translate で。

  • 控えめなら上品な立体感
  • 前庭系の酔い要因。reduced-motionで必ず停止。やり過ぎは“静か”に反する
難易度：中a11y注意

WCAG 2.3.3

⑥ 実装：IO ／ scroll()

On-enter（一度きり）vs Scrub

見せ場はIntersectionObserverで一度きりが最適。scrubはスクロール速度依存・逆再生・半端停止で“モーメント”が壊れる。

  • IO＝最軽量・自前クロックで確実に着地
  • CSS scroll-drivenはFirefox既定オフ→退行対策必須
難易度：低（IO）

Comeau ／ animation-trigger(’26)

⑦ 実装：per-line delay

テキスト 行ごと stagger

3行を少し遅らせて順に fade＋上昇。視線を下（メッセージ）へ誘導＝コピー主役の見せ場に最適。transition-delay だけ（ライブラリ不要）。

  • コピーが立つ／安価
  • 遅延を盛り過ぎると“もたつく”。合計~1.0–1.3s
難易度：低GPU◎

SplitText（無料）

⑧ 実装：overlay/filter

アンビエント（特別感の隠し味）

ビネット open／brightness ランプ（照明が灯る）／微グレイン／全体を scale .985→1 の“ひと呼吸”。

  • 静かに“特別”を足せる
  • アニメ・グレインと大blurは塗り負荷。控えめに
難易度：低〜中

Grain(SVG)

＋ イージング・尺・トリガー

イージング

到着して“収まる”減速カーブ。ease-out-expo cubic-bezier(0.16,1,0.3,1)（本命）／quint (0.22,1,0.36,1)／テキストは material (0.4,0,0.2,1)。bounce/linearは避ける。

尺

写真の登場 800–1100ms。各行 600–800ms＋行間 100–160ms（合計~1.0–1.3s）。Ken Burns 15–25s。~1200ms超は“もたつき”。

トリガー & a11y

IObserverで一度きり（threshold~0.3、発火後 unobserve）。prefers-reduced-motion では隠し状態を作らず即表示。parallax/大ズームを最優先で無効化。

推奨案

「静かでゆっくり」な見せ場に、最小工数で最大の“特別感”。すべて transform/opacity/filter＝軽量・依存ゼロ・退行安全。

1

写真：Blur-in＋Fade＋micro-Scale（IOで一度） ＋ テキスト：3行 stagger

写真が“ピントを結ぶ”ように立ち上がり（opacity0→1 / blur10→0 / scale1.06→1）、少し遅れて3行が順に浮かぶ。コピーが主役の見せ場に最適。任意で ビネット＋緩いKen Burnsで温度感を追加。

尺：写真 ~900ms ease-out-expo ／ 各行 ~700ms・行間 ~130ms（写真の~180ms後から）

2

Clip-path ワイプ（下→上）＋ Fade ＋ テキスト stagger

“シーンがめくれて現れる”編集的な見せ方。よりデザイン主張が強い。硬さを遅いeaseとfadeで和らげる。

3

Fade-up（尺長め）＋ テキスト stagger

最も安全・最小。既存 .appear の延長で確実。まず入れるならこれ。

※ Parallax／scroll-scrub は見せ場には不向き（逆再生・半端停止でモーメントが崩れる／酔い要因）。アンビエント用途に限定。

→ 本体（Astro）への落とし込み

  1. .appear--hero 変種を global.css に定義（opacity/filter/transform の隠し状態→.inで解除、ease-out-expo・~900ms）。既存の .appear/appear-observer.ts をそのまま再利用。
  2. MessageLead.astro の各行を <span class="msg-line"> 化し、transition-delay を 0/130/260ms（または --i 変数）で stagger。
  3. 写真レイヤーを .message::before から要素化 or 疑似要素に .appear--hero を適用（blur/scale/opacity）。appear-observer の threshold を ~0.3 に。
  4. reduced-motion：既存ブロックで .appear--hero も即表示（隠し状態を無効）。blur/scaleを切る。
  5. 任意アンビエント：ビネットのpseudo＋@keyframes Ken Burns（@media (prefers-reduced-motion: no-preference) 内）。
